Checksums

Integrity of downloaded packages

{% capture markdown %} _Checksums are short chunks of text which can verify that files were downloaded correctly. MyBB publishes checksums generated by `SHA-512`, `SHA-256`, `SHA-1` and `MD5` hash algorithms (strongest to weakest — checking `SHA-512` is recommended)._ After downloading the package, its checksum should be compared to the one that was published. - **Linux** — Terminal ```sh sha512sum filename.zip ``` - **Mac OS X** — Terminal ```sh shasum --algorithm 512 filename.zip ``` - **Windows** — Command Prompt ```sh CertUtil -hashfile filename.zip SHA512 ``` - **Windows** — PowerShell ```powershell Get-FileHash filename.zip -Algorithm SHA512 | Format-List ``` {% endcapture %} {{ markdown | markdownify }}

Reproducibility

Tracing the packaging process

{% capture markdown %} Distributed packages (`mybb_*.zip`, `changed_files_*.zip`) can be reproduced using: - the [build script](https://github.com/mybb/mybb-build), and - `build_*.zip` archives on the [Releases page on GitHub](https://github.com/mybb/mybb/releases), which include input for the script (`input/`): - references to the [MyBB source code repository](https://github.com/mybb/mybb) at a specific state to pull the code from, - security patches (`.diff` format), - metadata. The generated packages (`output/`) should be binary-identical to published packages. {% endcapture %} {{ markdown | markdownify }}

If you encounter discrepancies or notice suspicious content in any of the packages, contact the MyBB Security Team.
